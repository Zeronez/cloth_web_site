# Frontend Responsive Testing Targets

Цель: перед релизом прогнать основные сценарии (каталог, карточка товара, корзина, чек-аут, кабинет) на популярных размерах экранов.

## Мобильные (портрет)
- iPhone SE (375×667)
- iPhone 13/14/15 (390×844)
- iPhone 14/15 Pro Max (430×932)
- Android compact (360×800)
- Android large (412×915)

## Планшеты
- iPad mini / 8.3" (744×1133)
- iPad / 10.2" (810×1080)
- iPad Pro / 12.9" (1024×1366)

## Десктоп
- Ноутбук (1366×768)
- Full HD (1920×1080)
- Ultrawide (2560×1440)

## Чеклист
- Навигация и хедер не “прыгают” при скролле
- Кнопки доступны пальцем (tap targets) и не уезжают за экран
- Все формы читаемы, поля не перекрываются клавиатурой на мобильных
- Состояния: загрузка/ошибка/пусто отображаются корректно
- Фокус/клавиатурная навигация видимы (`Tab`)
