Уважаемый председатель и члены государственной аттестационной комиссии, вашему вниманию предлагается дипломный проект на тему: «Разработка информационной системы на базе фреймворка Django для интернет-магазина одежды». В рамках проекта разработан интернет-магазин аниме-одежды AnimeAttire.
Актуальность проекта связана с тем, что интернет-магазин одежды должен обеспечивать не только просмотр товаров, но и удобный подбор, корзину, оформление заказов и управление данными. Основные проблемы — большой объём товаров, размеров и цветов, риск ошибок в наличии, сложность выбора размера и ограниченность готовых платформ.
В ходе анализа были рассмотрены маркетплейсы, конструкторы интернет-магазинов и собственная разработка. Маркетплейс быстро запускается, но ограничивает бренд и пользовательский опыт. Конструктор удобен на старте, однако зависит от возможностей платформы. Собственная разработка сложнее, но обеспечивает гибкость, поддержку и полный контроль над развитием AnimeAttire.
Цель дипломного проектирования — разработка интернет-магазина аниме-одежды AnimeAttire для автоматизации подбора товаров, работы с каталогом, корзиной и заказами. Для достижения цели были выполнены анализ предметной области, формулирование требований, выбор технологий, проектирование архитектуры и базы данных, разработка API, интерфейса, формы-опроса и тестирование.
Для реализации проекта выбран стек технологий для клиент-серверной системы. Клиентская часть разработана на Next.js, React и TypeScript. Для состояния используются React Query и Zustand, для интерфейса — Tailwind CSS и Framer Motion. Серверная часть реализована на Python, Django 5 и Django REST Framework. Также используются JWT, OpenAPI, Swagger, PostgreSQL, Redis и Celery.
При сравнении технологий рассматривались React с Node.js, Django с Python и Spring с Java. React и Node.js гибкие, но требуют дополнительной настройки серверной части. Spring и Java мощные, но избыточны для данного проекта. Django и Python выбраны из-за скорости разработки, встроенной админ-панели, удобной работы с базой данных и API.
Архитектура системы построена по клиент-серверному принципу. Пользователь работает с frontend-приложением, которое отправляет запросы на backend. Frontend отвечает за интерфейс, а backend — за обработку данных, бизнес-логику, права доступа и взаимодействие с PostgreSQL. Внешние сервисы могут использоваться для интеграции доставки.
База данных спроектирована вокруг основных сущностей магазина: пользователей, товаров, категорий, вариантов товаров, корзины, заказов, адресов, избранного и данных формы-опроса. Такая структура позволяет хранить информацию связанно, контролировать наличие товаров и сохранять историю заказов. PostgreSQL обеспечивает целостность и надёжность данных.
Оформление заказа реализовано как последовательный сценарий. Пользователь выбирает товар, добавляет его в корзину, указывает данные и подтверждает заказ. Система проверяет корректность данных и наличие товаров, после чего создаёт заказ и сохраняет его состав. Для контроля обработки используются статусы заказа.
Пользовательский интерфейс включает главную страницу, каталог, карточки товаров, корзину, личный кабинет и форму-опрос. Главная страница знакомит с магазином, каталог помогает просматривать ассортимент, а форма-опрос учитывает размеры, стиль, бюджет и пожелания пользователя. Интерфейс адаптирован для компьютеров и мобильных устройств.
Доступ к базе данных выполняется только через backend. Сервер принимает запросы клиента, проверяет входные данные, обращается к моделям Django и возвращает результат через API. Это отделяет интерфейс от логики хранения данных и делает работу системы более безопасной и контролируемой.
Для проверки качества проведено тестирование системы. Системные проверки выполняются через GitHub Actions, frontend проверяется с помощью Jest, а backend — с помощью pytest. Автоматизированные тесты подтвердили корректность основных частей проекта: все проверки backend и frontend успешно пройдены без ошибок и пропусков.
Также выполнено функциональное тестирование интерфейса на примере регистрации нового пользователя. В тест-кейсе указаны предусловие, шаги, тестовые данные, ожидаемый и фактический результат. Проверка показала, что учётная запись создаётся корректно, пользователь проходит предусмотренный сценарий входа, а ошибки валидации отсутствуют.
В результате дипломного проекта разработан интернет-магазин AnimeAttire. Система обеспечивает просмотр каталога, подбор одежды по параметрам пользователя, работу с корзиной, оформление заказов и управление данными магазина. Разделение клиентской и серверной частей упрощает сопровождение и создаёт основу для дальнейшего развития.
В экономической части рассчитаны основные показатели: затраты на разработку составили 306 600 рублей 44 копейки, расчётная цена программного продукта — 486 268 рублей 30 копеек, чистая прибыль — 68 985 рублей 10 копеек. Можно сделать вывод, что цель проекта достигнута, а разработка является технически реализуемой и экономически обоснованной. Благодарю за внимание.
